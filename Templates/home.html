    ADMIN

    In this portal admin is a person who sells all the products. The admin maintains the website and uploads the details or list of the products which he sells. Admin will have access to all the information related to the products, manager and user. Admin will sell the products according to the requirements of the user. He will also have the list of managers and the users. Only admin can add the new manager into the portal. Admin can see the feedbacks from the manager and client.

    MANAGER

    In this portal manager is a person who acts as the bridge between the admin and the user. Manager also has the register and login credentials to enter into the portal. Using his login details he enters into the portal . Manager will have all the information about the list of products and list of customers in the portal. Manager also has the access to upload the products into the portal. If manager have any issues he can also give the feedback to the admin.

    USER

    In this portal user is a person who buys the products from the portal. User also needs to register and login for purchases. He can view the list of products, order details and he can also track his order. If user is facing any problems he can give the feedback that feedback is seen by the both manager and the user.

    This project is designed for the communication between the admin and the client. In this, every detail of the product is displayed to the clients. Manufacturers design several products and upload them to the site. So, those clients can know about the type, price, number, quality, and quantity of products.
    In this project, there are 3 modules which include:-
    Admin side:-In this admin is the owner of the portal. He adds the manager to the portal and maintains the portal.

    Client side:-In this every client should register /login into a site for further process. Clients can check the products and can book the products.
    Manager side:- This acts as a communication between both manufacturer and the client. Even every manager should register /login into a site for further process.
    If a client books the product, it should also show the tracking address of the product to the client. If a client wants to book a product that is out of stock then, this information should reach the admin and also to the manager.
    The database is connected between the admin side, manager side and the client side where all the transactions are recorded in it.
